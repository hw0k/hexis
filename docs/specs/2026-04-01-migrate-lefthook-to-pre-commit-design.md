# Migrate lefthook to Python pre-commit (uvx)

## Problem

The current lefthook-based git hook system is buggy and unreliable. Replace it with the Python `pre-commit` tool invoked via `uvx`.

## What

### Remove

- `lefthook.yml` (root)
- `.githooks/lefthook.yml` (copy)
- `.githooks/pre-commit`, `.githooks/commit-msg` (scripts generated by lefthook)
- lefthook installation instructions in `skills/setup-new-project/SKILL.md` and `CLAUDE.md`

### Add

`.pre-commit-config.yaml` with equivalent hook definitions:

```yaml
repos:
  - repo: local
    hooks:
      - id: lint
        name: lint
        language: system
        entry: .githooks/run-if-exists.sh lint
        stages: [pre-commit]
        pass_filenames: false
      - id: format
        name: format
        language: system
        entry: .githooks/run-if-exists.sh format
        stages: [pre-commit]
        pass_filenames: false
      - id: test
        name: test
        language: system
        entry: .githooks/run-if-exists.sh test
        stages: [pre-commit]
        pass_filenames: false
      - id: commitlint
        name: commitlint
        language: system
        entry: bunx commitlint --edit
        stages: [commit-msg]
        pass_filenames: true
```

### Installation Command

```bash
uvx pre-commit install
uvx pre-commit install --hook-type commit-msg
```

### Keep

- `.githooks/run-if-exists.sh` — continues as the entry point for local hooks

### Update

- `skills/setup-new-project/SKILL.md` — replace lefthook install/configure steps with uvx pre-commit
- `skills/use-worktree/SKILL.md` — add step to run `uvx pre-commit install` + `--hook-type commit-msg` after creating a worktree
- `CLAUDE.md` — remove/replace lefthook references

## Hooks Directory

pre-commit installs directly to `.git/hooks/` (version-controlled config lives in `.pre-commit-config.yaml`).

- Remove `core.hooksPath .githooks` git config (use pre-commit's default `.git/hooks/`)
- `.worktreeinclude` is no longer needed — each worktree requires a manual `uvx pre-commit install`

## Out of Scope

- No changes to `run-if-exists.sh` internals
- No changes to commitlint configuration (`.commitlintrc.yml`)
